Welcome to AgriClimate Connect API

The AgriClimate Connect API provides developers with programmatic access to the world's most comprehensive agricultural intelligence platform. Our RESTful API enables you to integrate climate-smart farming solutions, AI-powered crop advisory, real-time market data, and predictive analytics directly into your applications.

What You Can Build

Enterprise Dashboards

Base URL

All API requests should be made to the following base URL. We support both HTTP and HTTPS, though HTTPS is strongly recommended for production environments.

Base URL
https://api.agriclimate.com/v2

Quick Start Guide

Get up and running with your first API call in under 5 minutes. Follow these steps to make your first request to the AgriClimate Connect API.

1

Get Your API Key

Sign up for a developer account and generate your API key from the dashboard. Keep your API key secure and never expose it in client-side code.

2

Make Your First Request

Use cURL or your preferred HTTP client to test the API. Here's a simple example to get current weather data for a location:

cURL Example 
curl -X GET "https://api.agriclimate.com/v2/weather/current" \
 -H "Authorization: Bearer YOUR_API_KEY" \
 -H "Content-Type: application/json" \
 -d '{
   "latitude": -1.2921,
   "longitude": 36.8219
 }'
3

Handle the Response

Core Concepts

Geospatial Data

All location-based endpoints accept latitude/longitude coordinates with 4 decimal place precision. We automatically resolve coordinates to administrative regions for localized recommendations.

Temporal Ranges

Date parameters use ISO 8601 format. Forecast data is available up to 14 days forward, while historical data spans 40 years for climate trend analysis.

Data Granularity

Specify resolution from 1km (hyperlocal) to 25km (regional). Higher granularity requires Pro tier or above. Default is 5km for optimal coverage/accuracy balance.

Real-time Updates

Weather data updates every 15 minutes. Market prices refresh hourly during trading hours. Subscribe to webhooks for instant notifications on critical changes.

Data Formats

The API exclusively uses JSON for request and response bodies. All timestamps are returned in UTC and follow ISO 8601 format. Numeric values use standard JSON number types.

Field Type Description Example
datetime ISO 8601 timestamp "2026-03-08T14:30:00Z"
coordinate Decimal degrees, 4dp precision -1.2921
enum Predefined string values "optimal", "moderate", "poor"
measurement Value with unit object {"value": 24.5, "unit": "celsius"}
localized Multi-language string map {"en": "Maize", "sw": "Mahindi"}

Authentication

Secure your API requests using OAuth 2.0 or API key authentication. We recommend OAuth 2.0 for production applications and API keys for development and testing.

API Key Authentication

Simple header-based authentication for development

Include your API key in the Authorization header of every request. Keys can be generated and managed from your developer dashboard.

Header Format 
Authorization: Bearer ac_1234567890abcdef

Security Warning

Never expose your API key in client-side code or public repositories. Use environment variables and server-side proxying for production apps.

OAuth 2.0

Industry-standard authentication for production

We support the Authorization Code flow with PKCE for secure, token-based authentication. This is recommended for all production applications.

1. Authorization Endpoint

GET /oauth/authorize
 ?client_id={client_id}
 &response_type=code
 &redirect_uri={redirect_uri}
 &scope=weather:crops:market
 &state={random_state}
 &code_challenge={pkce_challenge}

2. Token Exchange

POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code
&code={authorization_code}
&redirect_uri={redirect_uri}
&client_id={client_id}
&code_verifier={pkce_verifier}
Token Response

Webhook Security

Verify webhook authenticity by checking the signature in the X-AgriClimate-Signature header. We use HMAC-SHA256 to sign payloads with your webhook secret.

Signature Verification (Node.js) 
const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
 const expected = crypto
   .createHmac('sha256', secret)
   .update(payload)
   .digest('hex');
 
 return crypto.timingSafeEqual(
   Buffer.from(signature),
   Buffer.from(expected)
 );
}
